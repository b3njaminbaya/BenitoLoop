# Phase 2 setup: transactional email

Three things need to happen once, then it's fully automatic.

## 1. Run the migration

Run `supabase/migrations/0016_transactional_email.sql` in the Supabase SQL editor.

Then run this and paste me the single value it returns (a random secret generated by the migration — it never appears in git):

```sql
select decrypted_secret from vault.decrypted_secrets where name = 'email_webhook_secret';
```

## 2. Get a Resend API key (free tier: 3,000 emails/month, no card required)

1. Sign up at resend.com.
2. Create an API key (Dashboard -> API Keys).
3. Paste me the key — I'll set it as an Edge Function secret via the Supabase CLI (`RESEND_API_KEY`), along with the webhook secret from step 1 (`EMAIL_WEBHOOK_SECRET`), then deploy the `send-email` function myself. Neither value is ever committed to the repo.

Order confirmations, donation acknowledgements, and partner application emails work as soon as this step is done — no further action needed.

## 3. Branded password reset (replaces Supabase's default email)

This part is Dashboard-only — Supabase doesn't expose SMTP/email-template config through the CLI or SQL.

1. In Resend, go to **Domains** and either verify your own domain, or skip this for now and use the shared `onboarding@resend.dev` sender (fine for testing, not for real users at scale).
2. In Supabase Dashboard -> **Project Settings -> Auth -> SMTP Settings**, enable custom SMTP:
   - Host: `smtp.resend.com`
   - Port: `465`
   - Username: `resend`
   - Password: your Resend API key
   - Sender email: `onboarding@resend.dev` (or your verified domain address)
   - Sender name: `Nyuzi`
3. In Supabase Dashboard -> **Authentication -> Emails -> Reset Password**, replace the default template with the contents of `supabase/email-templates/reset-password.html`.

Once SMTP is on, *all* Supabase auth emails (reset password, and email confirmation if you turn that on later) go out through Resend with Nyuzi branding instead of Supabase's default sender.

## Checking it worked

Admin panel -> **Email Log** shows every send attempt and its status (`sent`, `failed`, `skipped_no_email` for guest donations with no account, `skipped_not_configured` if step 2 hasn't happened yet).
